# 柯里化

柯里化主要就是把多参数的函数转换成但参数函数，单参数函数的好处在函数式中有所体现，这里不做细化

# 例1

如何把三参数函数 add(1,2,3) 变成：
curriedAdd(1)(2)(3)形式？

const curriedAdd = 
    a => b => c => add(a,b,c) // 就这么简单

# 例2

currify(addTwo)(1)(2) // 3
currify(addThree)(1)(2)(3) // 6
currify(addFour)(1)(2)(3)(4) // 10
实现这个currify

// 基本实现
function addThree(a,b,c) {
  return a+b+c;
}

const currify = (fn, params = []) =>{
  return (arg) => {
    params.push(arg)
    if(fn.length === params.length) {
      return fn(...params)
    } else {
      return currify(fn, [...params])
    }
  }
}
currify(addThree)(1)(2)(3); // 6

// 优化： 支持 currify(addFour)(1)(2,3)(4) 这种形式
const currify = (fn, params = []) =>{
  return (...args) => {
    if(fn.length === params.length + args.length) {
      return fn(...params, ...args)
    } else {
      return currify(fn, [...params, ...args])
    }
  }
}

// 简化代码：
const currify = (fn, params = []) => 
  (...args) => 
    fn.length === params.length + args.length
      ? fn(...params, ...args)
      : currify(fn, [...params, ...args])

# 其他使用场景

# 函数的组合：

function doubleSay(str) {
  return str + ', ' + str
}
function capitalize(str) {
  return str[0].toUpperCase() + str.substring(1)
}
function exclaim(str) {
  return str + '!'
}
let result = exclaim(capitalize(doubleSay('hello')))
console.log(result);  // Hello, hello

由于以上这种写法很难看，于是诞生出了管道

let result = "hello"
    |> doubleSay
    |> capitalize
    |> exclaim;
console.log(result);  // Hello, hello

这种语法很快就可以用了
但是现在想用的话可以用 Ramda.js

const say = R.compose(doubleSay,capitalize,exclaim)
say('hello'); // Hello, hello

细心的话会发现，这些函数都必须是单参数函数！ 如果这里的其中一个函数是多参数，管道就没用了
单参数函数在很多场景都很有用，柯里化 就是解决函数多参数转单参数的问题
