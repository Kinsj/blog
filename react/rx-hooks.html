# Rxjs, Rx-hooks

# 遇到问题

这次在开发公司的项目时遇到了请求数据时的竞态问题： 项目原型如下图：

当短时间执行多个加载动作时就会出现竞态问题，非常头疼！（到底加载更多，筛选请求对应数据等操作）

那么解决竞态问题最强大的自然是响应式编程思想的Rxjs啦！ 和react结合使用的话自然是引入 rx-hooks

rx-hooks 入门: 像呼吸一样自然：React hooks + RxJS

# 分析需求，建模问题

需求：

  1. 多次筛选时，只保留最后一次筛选，取消前面没有返回结果的筛选
  2. 到底加载时，如果有多次触发到底加载，在上次加载还没结束前不响应后面的到底事件
  3. 到底加载时出现筛选事件，直接取消到底加载事件

抽象一下：

  • 把两次个事件都抽象成加载事件，用触发时的标志区分事件类型，过滤出两条事件流（筛选事件流，到底加载事件流）。最后subscribe 这两条事件流
  • 在这两个事件流上可以各自实现 需求1 和 需求2
  • 这样在筛选事件流发出新值时阻断到底加载事件流上正在执行的操作就可以实现需求3

# 逐个实现

要实现两种事件发射到同一个事件流（加载事件流）就要使用useEventCallback()

# 过滤出两个加载事件流 （到底加载，筛选加载）

const [loadEventCallback] = useEventCallback(
    event$ => {

      // 1.筛选出两个加载事件流 （到底加载，筛选加载）
      const selectorChange$ = event$.pipe(
        filter(type => type === 'selectorChanged'),
      );

      const reachEnd$ = event$.pipe(
        filter(type => type === 'loadMore')
      );
    }
);

# 实现需求3

到底加载时出现筛选事件，直接取消到底加载事件 这个需求实际上只需要在loadMore$流的请求中加一句 takeUntil(selectorChange$)

# 完整代码

  const [loadEventCallback] = useEventCallback(
    event$ => {

      // 1.筛选出两个加载事件流 （到底加载，筛选加载）
      const selectorChange$ = event$.pipe(
        filter(type => type === 'selectorChanged'),
        // 筛选加载流发射事件时要设置 isNoMore 为 false
        tap(() => setIsNoMore(false)) // setState 两次的值相同不会触发重新渲染
      );

      const reachEnd$ = event$.pipe(
        filter(type => type === 'loadMore')
      );

      // 2.定义两个流的基本行为
      // 筛选流新事件阻断旧事件
      const reSearch$ = selectorChange$.pipe(
        map(() =>
          rxGetJsonApi('/getjob?' + getJobFormData(place)).pipe(
            retry(3),
            takeUntil(event$) // 请求过程中，事件流里出现选项变化事件则 abort
          )
        ),
        concatAll(),  // 序列化顺序，解决竞态
        tap(res => {
          if (res.code === 1 && res.data.list.length > 0) {
            dispatch({type: 'ResetJobOfPage', payload: {place, list: res.data.list}});
          } else {
            setIsNoMore(true);
          }
        })
      );

      const loadMore$ = reachEnd$.pipe(
        // 触发一次后 直到返回结果才触发第二次
        throttle(() =>
          rxGetJsonApi('/getjob?' + getJobFormData(place)).pipe(
            retry(3),
            // 筛选流新事件阻断到底加载流 takeUntil
            takeUntil(selectorChange$),  // 💡
            tap(res => {
              if (res.code === 1 && res.data.list.length > 0) {
                dispatch({type: 'PushJobOfPage', payload: {place, list: res.data.list}});
              } else {
                setIsNoMore(true);
              }
            })
          )
        ),
      );

      // 返回两个流的 subscribe
      return merge(reSearch$, loadMore$);
    }
  );

到这里，需求就全部实现啦！
