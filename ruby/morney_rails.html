# 注册

  1. rails g model User email:string password_digest:string
    创建（generate）一个 User 的 model（MVC的M），这个model包含 email 和 password_digest字段，并指定了字段类型。
    生成的是 User 表的迁移代码
  2. rails db:migrate 会执行之前生成的迁移代码，创建出user表
    表中包含了id email password_digest created_at updated_at 字段
    rails的强大之处就是用最少的代码实现最多的功能，很快就会有所体现
  3. 首先来实现密码加密功能has_secrue_password
    • 需要用到一个新的包bcrypt，去Gemfiles里取消注释gem 'bcrypt', '~> 3.1.7'
    • 在User.rb(model)里加入一句话 has_secure_password
      就实现了密码加密的功能

  2. 添加参数校验

  • 在 Model 里添加 validates 代码就可以对参数进行校验，如以下代码：
    # models/user.rb
class User < ApplicationRecord
  has_secure_password

  validates_presence_of :email  # 验证 email 必须存在
  validates_presence_of :password_confirmation, on: [:create] # 验证 password_confirm 必须 在create时 存在
  # 上面因为已经指定了 has_secure_password, 所以 :password 的存在一定会被校验，所以下面这里就不用加 :password了
  alidates_format_of :email, with: /.+@.+/, if: :email  # format_of 传入 参数名， with: 正则，就可以校验，只在传了email的情况下才做此校验
  validates_length_of :password, minimum: 6, on: [:create], if: :password  # length_of 传入 参数名， 最大/最小长度，只在传入password的情况下才做此校验
end
  1. 把结果返回给前端，render :json
  • 判断 user.save 的结果，正确则返回 json: user, status: 200。失败就返回 json: user.errors, status: 400
  • 贴出 user_controller.rb
    class UsersController < ApplicationController
  def create
    # params  # params 就是用户传过来的参数，可以打印看看
    user = User.new
    user.email = params[:email]
    user.password = params[:password]
    user.password_confirmation = params[:password_confirmation]
    if user.save
      render json: user, status: 200  # 如果 user.save 成功，返回这个user, 并添加状态码
    else
      render json: user.errors, status: 400  # 如果 user.save 失败（校验失败）,把错误返回给前台
    end
  end
end

  • 优化思路：
    • User.new 可以传参，{✉️ params[:email], :password: params[:password]...}
    • params.permit(:email, :password, :password_comfirmation) 可以表示 params 过滤出参数组成键值对
    • user.errors.empty? 可以检查是否有error，有则render errors，没有则render user
    • 如果先new 一个 model 再 save它。可以使用 create。 User.create
  • 优化结果：
    # 把 render_resource 方法提升到 application_controller里面，所有controller可以共享
# application_controller.rb
class ApplicationController < ActionController::API
  def render_resource(resource)
    if resource.errors.empty?
      render json: {resource: resource}, status: 200
    else
      render json: {errors: resource.errors}, status: 400
    end
  end
end

# users_controller.rb
class UsersController < ApplicationController
  def create
    render_resource User.create create_params  # 只需要这一行代码就可完成用户的创建和校验（校验实际是在model中完成）
  end

  def create_params
    params.permit(:email, :password, :password_confirmation) # ruby 不用写return
  end
end
