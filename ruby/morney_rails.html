# 数据库设计

# users表

  • 注册：邮箱、密码、确认密码
  • 数据库不保存明文密码，只存密文
  • 加密使用最佳实践 has_secure_password
  • 密文叫什么？最佳实践 password_digest
  • 注册成功之后使用mailer发欢迎邮件
  • 需要强制用户验证邮箱吗？可以强制，也可以不强制
    结论： users 表有两个字段 email 和 password_digest

# 注册

  1. rails g model User email:string password_digest:string
    创建（generate）一个 User 的 model（MVC的M），这个model包含 email 和 password_digest字段，并指定了字段类型。
    生成的是 User 表的迁移代码
  2. rails db:migrate 会执行之前生成的迁移代码，创建出user表
    表中包含了id email password_digest created_at updated_at 字段
    rails的强大之处就是用最少的代码实现最多的功能，很快就会有所体现
  3. 首先来实现密码加密功能has_secrue_password
    • 需要用到一个新的包bcrypt，去Gemfiles里取消注释gem 'bcrypt', '~> 3.1.7'
    • 在User.rb(model)里加入一句话 has_secure_password
      就实现了密码加密的功能
# 往数据库里添加一行记录的操作：  
u = User.new
u.email = '2@qq.com'
u.password = '123'
u.password_comfirmation = '123' # 故意输错使用 u.errors 查看错误信息
u.save  # 只需要调用 save 就会自动生成sql语句往数据库里添加一条记录
u.authenticate('123') # 验证密码可以获取用户所有信息，不用关心加密解密
# 修改
u.email = '3@qq.com'
u.save  # 数据库里的记录就会被修改
  1. 创建 controller
    • rails g controller users 命令创建 users controller
    • 创建增删改查路由，在routes.rb里添加以下代码
       get '/users', to: 'users#index' # 查全部
 get '/users/:id', to: 'users#show' # 查单个

  get '/hello', to: 'first#hello'  # get('/hello', {to: 'first#hello'}) 的缩写
  # 'first#hello' 代表 在 first 这个 controller 上 的 hello 方法 （controller 是 MVC 的 C）

users_controller.rb

class UsersController < ApplicationController
  def create
    # params  # params 就是用户传过来的参数，可以打印看看
    user = User.new
    user.email = params[:email]
    user.password = params[:password]
    user.password_confirmation = params[:password_confirmation]
    user.save
    p user.errors
  end
end
